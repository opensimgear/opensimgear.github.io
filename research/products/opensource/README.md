# Open-Source Component Research

Open-source and open-hardware project notes for simulator hardware categories covered by
`src/content/docs/docs/components`.

Each project file should use these fields where discoverable:

- `project`
- `maintainer_or_org`
- `component_category`
- `subcategory`
- `maturity_or_status`
- `license`
- `repo_url`
- `docs_url`
- `source_urls`
- `key_features`
- `hardware_or_bom`
- `firmware_or_software_stack`
- `build_requirements`
- `compatibility`
- `strengths`
- `limitations`
- `commercial_analogs`
- `fit_notes`
- `last_checked`

Mark uncertain values with `[uncertain]`. Prefer official repositories, project docs, release pages, and published
CAD/BOM sources. Use forum threads, videos, and build logs only when official documentation is thin.
