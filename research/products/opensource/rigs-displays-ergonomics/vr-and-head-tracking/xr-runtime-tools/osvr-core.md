# OSVR Core

- project: OSVR Core
- maintainer_or_org: OSVR / Sensics and contributors
- component_category: vr-and-head-tracking
- subcategory: VR/AR software platform
- maturity_or_status: Dormant legacy platform; developer portal generated in 2018 and latest captured OSVR-Core release
  from 2015.
- license: Apache-2.0
- repo_url: https://github.com/OSVR/OSVR-Core
- docs_url: https://osvr.github.io/
- source_urls:
  - https://osvr.github.io/
  - https://github.com/OSVR/OSVR-Core
- key_features:
  - Open-source software platform for discovery, configuration, and operation of VR/AR devices and peripherals.
  - Core libraries, applications, and plugins for OSVR software stack.
  - Developer portal describes support for multiple engines, operating systems, asynchronous time warp, and direct mode.
- hardware_or_bom:
  - Software platform; no new hardware BOM.
  - Historically associated with OSVR Hacker Development Kit and supported peripherals.
- firmware_or_software_stack:
  - C++ core libraries, ClientKit, PluginKit, apps, plugins, CMake build, vendored dependencies.
- build_requirements:
  - C++ build environment and CMake.
  - Legacy docs/build paths may need modernization.
- compatibility:
  - Historical Unity/Unreal/VRPN-style VR integrations.
  - Modern OpenXR/SteamVR stacks have largely superseded it.
- strengths:
  - Apache-2.0 license.
  - Useful historical reference for open VR abstraction layers.
  - Shows plugin-based device framework ideas.
- limitations:
  - Dormant and likely not suitable for new sim builds.
  - Device ecosystem and runtime support are outdated.
  - Open software does not make all associated hardware fully open.
- commercial_analogs:
  - SteamVR runtime
  - OpenXR runtimes
  - Vendor VR SDKs
- fit_notes:
  - Include for completeness in open VR corpus, but steer new projects toward Monado/OpenXR.
  - Best value is legacy code/reference, not end-user simulator setup.
- last_checked: 2026-04-29
