# OpenHMD

- project: OpenHMD
- maintainer_or_org: OpenHMD community
- component_category: vr-and-head-tracking
- subcategory: open HMD driver API
- maturity_or_status: Unmaintained; README says OpenHMD is not actively maintained and XR driver work has largely moved
  to Monado.
- license: Boost Software License 1.0
- repo_url: https://github.com/OpenHMD/OpenHMD
- docs_url: http://www.openhmd.net/
- source_urls:
  - https://github.com/OpenHMD/OpenHMD
  - http://www.openhmd.net/index.php/download/
  - https://github.com/OpenHMD/OpenHMD/wiki/Support-List
  - https://monado.freedesktop.org/
- key_features:
  - Free/open-source API and drivers for immersive technology such as head-mounted displays with built-in head tracking.
  - Supports multiple platforms: Linux, Windows, OS X, Android, FreeBSD.
  - Includes drivers for Rift, Deepoon, PSVR, Vive, Nolo, WMR, XGVR, VR-Tek, external, and Android paths.
  - Provides language bindings in Go, Java, .NET, Perl, Python, and Rust through community projects.
- hardware_or_bom:
  - No new headset hardware; driver/API layer for existing HMDs.
  - Needs compatible USB/HID HMDs and platform-specific udev/setup on Linux.
- firmware_or_software_stack:
  - C library with Meson, CMake, or autotools build paths.
  - HIDAPI and driver modules.
- build_requirements:
  - Meson/Ninja, CMake, or autotools, HIDAPI, platform compiler.
  - udev rules on Linux for user access to devices.
- compatibility:
  - Useful for unsupported/older HMDs and as a source of open drivers.
  - Monado now leverages OpenHMD community drivers for additional HMD support.
- strengths:
  - Permissive license.
  - Important historical base for open HMD reverse-engineering.
  - Still useful as driver reference code.
- limitations:
  - Explicitly unmaintained.
  - Limited current runtime/compositor functionality compared with Monado/OpenXR.
  - Device support varies by HMD and platform.
- commercial_analogs:
  - Vendor VR runtimes/drivers from Meta, Valve, HTC, Microsoft
  - SteamVR driver stack
- fit_notes:
  - Use as reference or dependency through Monado rather than as a primary 2026 runtime choice.
  - Worth tracking for open-source support of legacy headsets in sim setups.
- last_checked: 2026-04-29
