# Splash

- project: Splash
- maintainer_or_org: Emmanuel Durand / SAT Metalab / Lab148 contributors
- component_category: display-systems
- subcategory: projection warping and blending software
- maturity_or_status: Active-ish open-source project; GitHub mirror says development moved to GitLab; latest captured
  release 0.10.8 from 2024.
- license: GPL-3.0-or-later
- repo_url: https://gitlab.com/splashmapper/splash
- docs_url: https://splashmapper.xyz/
- source_urls:
  - https://github.com/paperManu/splash
  - https://splashmapper.xyz/
  - https://projection-mapping.org/splash/
- key_features:
  - Modular multi-projector video-mapping software.
  - Calibrates projector intrinsic/extrinsic parameters, blending, and color from UV-unwrapped 3D display surfaces.
  - Handles multiple inputs and 3D models; tested up to eight outputs on two graphics cards.
  - Supports video files, cameras/capture cards, NDI, shmdata, and Blender-assisted configuration.
- hardware_or_bom:
  - Requires one or more projectors, a calibrated/UV-mapped display surface model, GPU outputs, and optional cameras for
    calibration.
  - Works for fulldomes, curved surfaces, buildings, moving objects, and other projection surfaces.
- firmware_or_software_stack:
  - C++/OpenGL application using FFmpeg, GSL, GLFW, GLM, ZeroMQ, JsonCpp, ImGui, optional Python, GPhoto, PortAudio,
    OpenCV, and shmdata.
- build_requirements:
  - Linux/Windows build dependencies per README, CMake/Ninja or packages.
  - Blender or other 3D tool to prepare UV-unwrapped surface geometry.
  - Projection calibration workflow knowledge.
- compatibility:
  - Relevant to simulator projection domes, cylindrical screens, and curved multi-projector visuals.
  - Not a direct plugin for racing/flight sims; it maps video/render outputs after content generation.
- strengths:
  - Purpose-built for multi-projector calibration, blending, and color correction.
  - More simulator-relevant than simple quad mappers because it supports 3D surface models.
  - GPL source can be inspected and modified.
- limitations:
  - Setup complexity is high.
  - Requires 3D model and projector calibration knowledge.
  - Single-computer limitation noted in README, with multi-computer support only planned.
- commercial_analogs:
  - Fly Elise-ng Immersive Display PRO
  - VIOSO Anyblend/Integrate
  - Warpalizer
  - Pixelwix
- fit_notes:
  - Strong open candidate for dome or cylindrical projection experiments.
  - For casual triple-monitor racing, native simulator triple-screen support is much simpler.
